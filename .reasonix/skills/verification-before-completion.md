# Verification Before Completion

> 声称工作已完成前，必须先执行验证。**核心原则：证据先于断言。**
> 来源：Trae skills / `verification-before-completion`

## 铁律

```
没有新鲜的验证证据，就不能声称通过。
```

## 验证门函数

```
声明任何状态之前：

1. 识别：什么命令能证明这个声明？
2. 执行：运行完整的命令
3. 读取：完整输出，检查退出码
4. 验证：输出是否确认声明？
   - 否 → 报告实际状态（附证据）
   - 是 → 声明状态（附证据）
5. 然后才能：做声明
```

## 常见验证对照表

| 声明 | 需要验证 | 不充分 |
|------|---------|--------|
| 测试通过 | 测试命令输出: 0 failures | 之前跑过、"应该没问题" |
| Lint 干净 | Lint 输出: 0 errors | 部分检查、推测 |
| 构建成功 | 构建命令: exit 0 | Lint 通过、log 看起来好 |
| Bug 已修 | 重现原始症状: 通过 | 代码改过了、假设修好了 |
| 需求满足 | 逐行核对清单 | 测试通过 |
