# Brainstorming Ideas Into Implementation

> 创意工作前必须先 Brainstorm。**禁止在未呈现设计方案并获用户批准前写代码。**
> 来源：Trae skills / `brainstorming`

## 流程

### Step 1: 探索项目上下文
- 检查相关文件、文档、最近的提交

### Step 2: 问澄清问题
- 每次问一个，理解目的/约束/成功标准

### Step 3: 提出 2-3 种方案
- 带优缺点分析和推荐

### Step 4: 呈现设计方案
- 按复杂度缩放，每节获得用户批准后再继续

### Step 5: 写设计文档
- 保存到 `Doc/superpowers/specs/YYYY-MM-DD-<topic>-design.md`

### Step 6: 过渡到实施
- 调用 `writing-plans` skill 创建实施计划

## 硬门禁
```
在呈现设计并获得用户批准之前，不得调用任何实施 skill、写代码或搭建项目。
即使看起来很简单也必须走这个流程。
```
