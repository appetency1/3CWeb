# Karpathy Guidelines

> 减少 LLM 常见编码错误的行为准则，源自 Andrej Karpathy 的观察。
> 来源：Trae skills / `karpathy-guidelines`

## 1. 先思考再编码
- 明确说出你的假设，不确定就问
- 存在多种解读时，展示出来不要暗选
- 有更简单的方案就说出来

## 2. 简单优先
- 只写解决问题的最小代码量
- 不写没要求的功能
- 不为单次使用创建抽象层
- 不写"灵活性/可配置性"的冗余代码
- 200 行能缩写成 50 行就重写

## 3. 外科手术式修改
- 只碰必须改的部分
- 不要"顺手"改进相邻代码/注释/格式
- 不要重构没坏的东西
- 匹配现有风格，即使你不那么做
- 发现无关的 dead code 只提一句，不删

## 4. 边界的明确
- 每个假设都要质疑
- 空值、边界情况、并发、权限

## 5. 可验证的成功标准
- 写完就说"完成了"之前：如何证明？
- 让测试定义"完成"
