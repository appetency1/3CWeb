# Doc Co-Authoring Workflow

> 协同编写文档、提案、技术规格的三步工作流。
> 来源：Trae skills / `doc-coauthoring`

## 三步流程

### Stage 1: 背景收集
提问收集元背景：
1. 这是什么类型的文档？（技术规格、决策文档、提案等）
2. 主要读者是谁？
3. 希望读者看完后有什么反应？
4. 有没有模板或格式要求？
5. 其他约束或背景？

### Stage 2: 精炼与结构
- 逐节构建，通过迭代编辑打磨
- 每节完成后经用户确认再继续
- 保持文档结构清晰、内容准确

### Stage 3: 读者测试
- 以全新上下文阅读文档
- 检查：新手读者能理解吗？有没有盲点？
- 修复所有发现的歧义和缺失

## 文档类型模板

### 决策记录 (ADR)
```
# ADR N: [标题]
- 日期: YYYY-MM-DD
- 状态: [提议/接受/废弃]
- 上下文: [背景和动机]
- 决策: [做了什么选择]
- 理由: [为什么选这个]
- 后果: [正面/负面影响]
```

### 技术规格
```
# [功能] 技术规格
- 目标
- 方案对比
- 详细设计
- API 变更
- 数据模型变更
- 兼容性
```
