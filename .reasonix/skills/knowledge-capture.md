# Knowledge Capture

> 将对话、讨论和见解转化为结构化文档记录。
> 来源：Trae skills / `knowledge-capture`

## 内容识别

从对话中提取：
- 关键概念和定义
- 做出的决策及理由
- 操作步骤和流程
- 重要见解或经验
- Q&A 配对
- 示例和使用场景

## 内容分类

| 类型 | 说明 |
|------|------|
| 概念/定义 | 术语解释、系统概念 |
| 操作指南 (How-to) | 步骤流程、最佳实践 |
| 决策记录 (ADR) | 决策 + 理由 + 后果 |
| FAQ | 常见问题解答 |
| 会议纪要 | 讨论要点+行动项 |
| 复盘/经验 | 学到的东西 |
| 参考文档 | API、配置、数据模型 |

## 存储位置

记录保存到对应位置：
- 项目决策 → 追加到 `Doc/` 相关文档或 `ARCHITECTURE.md`
- 技术方案 → `Doc/migrations/` 或 `Doc/` 下独立文档
- 日常经验 → 使用 `remember` 工具保存为项目 memory

## 约束
- 记录的决策优先存为 ADR 格式
- 每次会议/讨论后提取行动项
- 保持文档与代码同步
