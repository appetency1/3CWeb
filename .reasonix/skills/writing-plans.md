# Writing Plans

> 在接触代码前，为多步骤任务编写完整的实施计划。
> 来源：Trae skills / `writing-plans`

## 计划文档结构

### 文件映射
- 列出将要创建/修改的所有文件
- 每个文件的职责说明
- 按职责而非技术层拆分

### 步骤粒度
每个步骤应是一个独立动作（2-5分钟）：
- "编写失败测试" — 一个步骤
- "运行确认失败" — 一个步骤
- "实现最小代码让测试通过" — 一个步骤
- "验证测试通过" — 一个步骤

### 计划头部
每个计划以这个头部开始：
```
# [功能名称] 实施计划

- **产品/功能名称:** [名称]
- **日期:** YYYY-MM-DD
- **构建者:** Reasonix Code
- **审阅人:** [用户]
```

### 节结构
每节包含：
```
## 节 N: [标题]

### 目标
[一句话]

### 上下文
[背景信息]

### 边界
[范围是什么 / 不是什么]

### 实施步骤
1. [步骤] — [文件]
2. [步骤] — [文件]
[...]

### 验证
[如何测试]
```
