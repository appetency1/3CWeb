## 1. 修了什么

### 1.1 数据库：MySQL 9.2 授权
- **症状**：`Access denied for user 'root'@'127.0.0.1' to database 'clothes_demo'`
- **原因**：`root@127.0.0.1` 这条 user 记录**没被授权**访问 `clothes_demo` 库；`root@localhost` 才有权限
- **修复**：
  ```sql
  GRANT ALL PRIVILEGES ON clothes_demo.* TO 'root'@'127.0.0.1';
  GRANT ALL PRIVILEGES ON clothes_demo.* TO 'root'@'localhost';
  FLUSH PRIVILEGES;
  ```
- **验证**：`mysql -h 127.0.0.1 -u root -p000000 -e "USE clothes_demo; SHOW TABLES;"` → 返回 12 张表

> ⚠️ 提示：jdbc url 写的是 `localhost`，JDBC 会按 `127.0.0.1` 走协议，所以必须授权 `@'127.0.0.1'`。

### 1.3 前端：`http` 不能当函数用
- **症状**：5 个 API 全报 `http is not a function`
- **原因**：`src/utils/request.ts` 里 `http` 被定义成普通对象 `{ get, post, put, del }`，但 `src/api/*.ts` 里全都写成 `http<T>(url, ...)` 当函数调 — 6 个 api 文件全错
- **修复**：把 `http` 改成**可调用对象**，默认 GET：
  ```ts
  const httpFn = <T = unknown>(url, params?, methodOrCfg: Method | AxiosRequestConfig = 'GET') => {
    // method 为 'GET' 时 params→query；非 GET 时 params→body
    // methodOrCfg 为字符串当方法，传对象当 AxiosRequestConfig
    ...
  }
  export const http = Object.assign(httpFn, { get, post, put, del })
  ```
  这样**6 个 api 文件一行没改**。
