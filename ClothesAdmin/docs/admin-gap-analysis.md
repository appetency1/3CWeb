# 管理后台功能缺口分析

> 审计依据：数据库表结构 + 后端 API 覆盖 + 前端页面绑定

## 🔴 P0 — 后端有 API，前端完全没页面

| # | 功能 | 后端 API 已有 | 前端现状 |
|---|------|-------------|---------|
| 1 | **SKU 管理** | GET/POST/PUT/DELETE `/api/admin/sku/{goodsId}` | ❌ 无页面。商品规格（颜色/尺码/各规格价格库存）没法管理 |
| 2 | **订单详情** | GET `/api/admin/order/{id}` 返回完整订单+商品+地址 | ❌ 无详情页，点订单只能看到一行列表 |
| 3 | **用户状态切换** | PUT `/api/admin/user/{id}/status` | ❌ 列表只有查看，没有启用/禁用按钮 |
| 4 | **用户详情** | GET `/api/admin/user/{id}` | ❌ 没有详情弹窗或页面 |
| 5 | **修改密码** | POST `/api/admin/password` (AdminServlet 已有) | ❌ 设置菜单没有对应入口 |

## 🟠 P1 — 前后端都有但体验不完整

| # | 功能 | 问题 |
|---|------|------|
| 6 | **商品表单** (GoodsForm) | 没有 SKU 编辑界面（后端会接收 skus 数组但前端只提交基础字段） |
| 7 | **图片上传** | 所有图片（轮播图、商品封面）都是手动填 URL。**后端已有 `POST /api/public/upload`**（multipart，支持 jpg/png/webp，存到 `~/clothes-uploads/`，返回 URL），前端 GoodsForm、BannerList 均未对接 |
| 8 | **数据筛选** | 订单列表无法按关键词搜索（后端 `/api/admin/order?keyword=` 已实现）；商品列表无法按分类筛选（后端 `?categoryId=` 已实现） |
| 9 | **分页** | 用户列表、分类列表、轮播图列表全量加载，数据多了会卡 |

## 🟡 P2 — UI 风格不统一

| # | 页面 | 问题 |
|---|------|------|
| 10 | **商品列表** (GoodsList) | 仍是 Vant 移动端风格，跟新设计的仪表盘风格不搭 |
| 11 | **订单列表** (OrderList) | 同上，Vant 风格 |

## 🔵 P3 — 偏业务的功能缺失

| # | 功能 | 说明 |
|---|------|------|
| 12 | **评价管理** | 数据库有 `comment` 表，后端有评价相关 API，但没有后台评价审核/查看页面 |
| 13 | **数据导出** | 订单/商品列表没有导出 Excel/CSV 功能 |
| 14 | **管理员多角色** | 数据库 `admin` 表没有角色字段，只有单管理员 |
| 15 | **操作日志** | 没有记录谁在什么时候改了什么的审计日志 |
